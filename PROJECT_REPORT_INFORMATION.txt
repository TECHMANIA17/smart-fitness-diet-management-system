SMART FITNESS AND DIET MANAGEMENT SYSTEM - PROJECT REPORT INFORMATION

1. Project Title

Smart Fitness and Diet Management System


2. Project Overview

Smart Fitness and Diet Management System is a dynamic full-stack web application developed for users who want to manage fitness, diet, BMI, hydration, workout plans, and daily progress in one place. The system also includes an admin panel for managing users, diet plans, workout plans, exercises, tips, and progress records.

The project is suitable for a college mini project because it demonstrates user authentication, admin authentication, CRUD operations, REST API design, MongoDB database handling, personalized health calculations, and deployment support using Netlify.


3. Problem Statement

Many users find it difficult to track their health details, diet plans, workouts, BMI, water intake, and progress manually. Gyms, trainers, and diet consultants also need a simple platform to manage fitness plans and monitor user progress.

This project solves the problem by providing a centralized web-based system where users can calculate health metrics, receive personalized diet and workout suggestions, and track progress. Admins can maintain the content and manage user records through a dedicated admin dashboard.


4. Objectives

- To provide a user-friendly fitness and diet management system.
- To allow users to register, log in, and manage their profile.
- To calculate BMI, BMI category, ideal weight range, recommended calories, and water intake.
- To suggest personalized diet plans and workout plans based on user data.
- To allow users to record and view fitness progress.
- To provide an admin panel for managing users, diet plans, workouts, tips, exercises, and progress.
- To store data securely using MongoDB.
- To use JWT authentication and password hashing for security.

11. Health Calculation Logic

BMI Calculation:
- BMI = weight in kilograms / height in meters squared
- The app calculates BMI using heightCm and weightKg.

BMI Categories:
- Below 18.5: Underweight
- 18.5 to 24.9: Normal
- 25 to 29.9: Overweight
- 30 and above: Obese

Ideal Weight Range:
- Minimum ideal weight is calculated using BMI 18.5.
- Maximum ideal weight is calculated using BMI 24.9.

Calorie Recommendation:
- The app calculates BMR using user gender, age, height, and weight.
- It multiplies BMR with activity level.
- It adjusts calories based on fitness goal.

Water Intake:
- The app recommends daily water intake using user weight.


12. Functional Flow

1. User opens the website.
2. User registers with name, email, password, and fitness profile data.
3. Password is hashed before saving.
4. User logs in and receives a JWT token.
5. User dashboard displays health data and recommendations.
6. BMI, ideal weight, calories, and water intake are calculated automatically.
7. User views diet and workout plans.
8. User records daily progress.
9. Admin logs in through the admin panel.
10. Admin manages users, diet plans, workout plans, exercises, tips, and progress records.


13. Security Features

- Passwords are hashed using bcryptjs.
- JWT is used for user and admin authentication.
- User routes are protected using user authentication middleware.
- Admin routes are protected using admin authentication middleware.
- Environment variables are used for MongoDB connection string and JWT secret.
- Separate user and admin access control is implemented.


14. Project Advantages

- Simple and easy-to-use interface.
- Separate dashboards for users and admins.
- Personalized health calculations.
- Dynamic data management using MongoDB.
- Complete CRUD functionality.
- Suitable for gyms, trainers, diet consultants, and health centers.
- Beginner-friendly structure for academic project submission.
- Can be deployed using Netlify.


15. Limitations

- The system provides general fitness and diet recommendations, not medical advice.
- It requires internet and database connectivity for full backend functionality.
- Advanced AI-based recommendation is not implemented.
- Payment, appointment booking, and trainer chat features are not included.

17. Conclusion

The project demonstrates important concepts such as frontend development, backend API development, database management, authentication, authorization, health calculations, CRUD operations, and deployment support. 19. How to Run Locally

1. Open the project folder in terminal.
2. Go to the backend folder if running backend scripts directly.
3. Create a .env file with MongoDB connection and JWT secret.
4. Install dependencies using npm install.
5. Optional: add sample data using npm run seed:sample.
6. Start the server using npm start.
7. Open http://localhost:5000 in the browser.
